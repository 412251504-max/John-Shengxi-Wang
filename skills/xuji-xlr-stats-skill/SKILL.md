# 薛记学习率统计 Skill

## 概述

本 skill 将以下手动操作流程自动化：
1. 从学习系统导出的 **项目地图情况表** 中提取数据
2. 剔除 **员工账号管理** 中的停用账号
3. 只保留门店一线人员（营业员、店经理、店经理助理等）
4. 结合 **月架构** 表补充小区负责人信息
5. 输出三份标准 Excel 报表

处理脚本位于 `scripts/process.py`，所有核心逻辑已固化。

---

## 输入文件识别

| 文件特征 | 对应输入 |
|---------|---------|
| 含"项目地图情况表" 或 sheet 名含该字样 | 项目地图情况表（学习数据主表） |
| 含"员工账号管理" 或 sheet 名含该字样 | 需剔除人员名单 |
| 含"架构" 且有 Sheet/Sheet1 两个 sheet | 月架构表（门店-区域映射） |

用户可能使用不同月份的文件（如"4月架构.xlsx"），识别时注意灵活匹配。

---

## 核心处理逻辑

### 1. 人员过滤规则

**剔除条件（以下任一）：**
- 员工编号在「员工账号管理」表中（停用账号）
- 部门名称（直属门店字段）不是门店编码格式——即不以「字母 + 3位以上数字」开头
  - 例：`H562-南通通州南山湖店` → **保留**（门店一线员工）
  - 例：`财务部`、`综合管理部`、`南通一区`、`合伙人成功三部` → **剔除**（非门店人员）

**过滤后保留的典型岗位：** 营业员、店经理、店经理助理、储备店经理、训练员、训练组长等

### 2. 小区负责人匹配

通过「月架构」Sheet 的 `机构编码` 列与员工的门店编码做精确匹配。  
架构表中有4位小区负责人，对应4个成功部：

| 小区负责人 | 成功部 | 覆盖范围 |
|-----------|-------|---------|
| 周亭 | 合伙人成功一部 | 苏皖鄂陕晋渝 |
| 赵丹 | 合伙人成功二部 | 浙赣 |
| 岳辉 | 合伙人成功三部 | 沪苏通 + 上海直营 |
| 王刚 | 合伙人成功四部 | 南京合伙 + 南京直营 |

> ⚠️ **人员变动提示**：如小区负责人发生调整，需更新 `scripts/process.py` 中的 `XQ_TO_BU` 字典。

### 3. 学习率计算

- `完成状态 == '已完成'` → 计入「已完成人数」
- 其余（未开始、进行中）→ 计入「未完成人数」
- `学习率 = 已完成人数 / 总人数`

### 4. 直营 vs 合伙 区分

- `部门名称3 == '东区事业部'` → 直营人员
- 其余 → 合伙人员

直营人员的成功部归属直接取 `部门名称4` 字段（已包含"合伙人成功X部"）。

## 新架构表适配

每月架构表更新时：
- Sheet 名为 `Sheet`（含完整门店数据，有 `机构编码` 列）的 sheet 用于主要匹配
- 如架构表中有新门店（报告期内新开门店），其员工会通过「区域降级匹配」自动归入正确区域
- 如小区负责人变更，需手动更新 `process.py` 中的 `XQ_TO_BU` 字典

## 常见问题

**Q: 输出人数比上月少？**  
A: 检查员工账号管理名单是否有更新，或项目地图表是否包含了所有人员。

**Q: 某区域学习率明显异常？**  
A: 查看单项目完成情况表，筛选该区域，检查是否有大量"未开始"状态。

**Q: 新门店人员未统计进去？**  
A: 新门店代码应已进入项目地图，但若架构表未更新，其小区负责人会通过区域降级匹配推算。确认无误后可更新架构表。

**Q: 成功部分组有误？**  
A: 检查 `process.py` 中 `XQ_TO_BU` 字典与当前小区负责人是否一致。
